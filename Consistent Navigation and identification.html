WCAG 3.2 – Consistency

This page demonstrates consistent navigation and consistent identification with passing and failing examples.

3.2.3 – Consistent Navigation

✅ Passing Example (Navigation is Consistent)

Why this PASSES:
  • Navigation appears in the same order
  • Same labels used across pages
  • Position remains consistent
Home Products Support Contact

This navigation would appear the same on every page.

❌ Failing Example (Navigation is Inconsistent)

Why this FAILS:
  • Navigation order changes
  • Labels are different for the same destination
Home Products Help
Start Help Shop

Users cannot rely on navigation location or labels being consistent.

3.2.4 – Consistent Identification

✅ Passing Example (Controls Identified Consistently)

Why this PASSES:
  • Same control uses the same label everywhere
  • No confusion for screen reader or cognitive users

This "Search" button is labeled the same across all pages.

❌ Failing Example (Controls Identified Inconsistently)

Why this FAILS:
  • Same function uses different labels
  • Creates confusion for assistive technology users

Both buttons perform the same action but are identified differently.

Accessibility Tip:
When navigation mechanisms or components are repeated, they must appear and be identified consistently unless a change is initiated by the user.
